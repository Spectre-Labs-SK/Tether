**If lint fails:**

Common causes and fixes:

1. `noUnusedLocals` / `noUnusedParameters` error — a variable was added but not used. Remove the unused variable or add it to the dependency array.

2. `verbatimModuleSyntax` error — a type is imported without `import type`. Fix by changing `import { SomeType }` to `import type { SomeType }` for type-only imports. Note: `import { useEffect }` is a value — correct. `import { UIConfig }` when UIConfig is only used as a type — change to `import type { UIConfig }`.

3. `mat.speed` TypeScript error — MeshDistortMaterial may not expose `speed` as a mutable property in its type definition. If TypeScript complains, cast: `(mat as unknown as { speed: number; distort: number }).speed = ...`. Apply the same cast for distort if needed. This is a three/drei type gap, not a logic error.

4. React hooks exhaustive-deps warning about `trickycardio` / `bitchweights` — these are `useCallback`-wrapped in useTetherState. Include them in the useEffect dependency array in WarRoom. They are stable references.

5. ESLint react-refresh plugin warning about non-component exports — if usePatternObserver or patternStore triggers this, it is a warning not an error; the build will still succeed.

**If build fails after lint passes:**

Check for import resolution errors first:
```bash
ls /Users/cjsander/Spectre_Labs/Tether/node_modules/zustand/index.js 2>/dev/null || echo "MISSING"
ls /Users/cjsander/Spectre_Labs/Tether/src/stores/patternStore.ts 2>/dev/null || echo "MISSING"
ls /Users/cjsander/Spectre_Labs/Tether/src/hooks/usePatternObserver.ts 2>/dev/null || echo "MISSING"
```

Fix any missing files before re-running build.

**Success state:** Both commands exit 0 with no errors. Warnings are acceptable — errors are not.
