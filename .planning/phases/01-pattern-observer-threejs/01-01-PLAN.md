Completely replace the contents of src/components/ShimmerCore.tsx. The old file accepted props (mode, staticLevel) — the new version accepts zero props and reads entirely from usePatternStore.

⚠ EXECUTOR NOTE: RESEARCH.md contains a code sample that uses `colorRef.current.lerp(new THREE.Color(target.color), LERP)` — do NOT follow it. That pattern allocates a new THREE.Color on every frame (GC pressure at 60fps). The correct pattern is below using the pre-allocated `_targetColor` module-level constant.

Write exactly:

```typescript
import { useRef } from 'react';
import { useFrame } from '@react-three/fiber';
import { MeshDistortMaterial, Float } from '@react-three/drei';
import * as THREE from 'three';
import { usePatternStore } from '../stores/patternStore';

const LERP = 0.04;

// Pre-allocate a scratch Color so useFrame never allocates new THREE.Color() at 60fps.
const _targetColor = new THREE.Color('#1e293b');

export function ShimmerCore() {
  const materialRef = useRef<InstanceType<typeof MeshDistortMaterial>>(null!);

  // floatIntensity and floatSpeed are React props on <Float> — they cannot be
  // mutated in useFrame. Read them reactively here (outside Canvas render loop)
  // so <Float> re-renders only on domain transitions, not every frame.
  const floatIntensity = usePatternStore((state) => state.target.floatIntensity);
  const floatSpeed = usePatternStore((state) => state.target.floatSpeed);

  useFrame(() => {
    if (!materialRef.current) return;
    // Use getState() (non-reactive) — never call usePatternStore() inside useFrame.
    const { target } = usePatternStore.getState();
    const mat = materialRef.current;

    mat.distort = THREE.MathUtils.lerp(mat.distort, target.distort, LERP);
    mat.speed = THREE.MathUtils.lerp(mat.speed ?? 2, target.speed, LERP);
    mat.metalness = THREE.MathUtils.lerp(mat.metalness, target.metalness, LERP);

    _targetColor.set(target.color);
    mat.color.lerp(_targetColor, LERP);
  });

  return (
    <Float speed={floatSpeed} rotationIntensity={0.5} floatIntensity={floatIntensity}>
      <mesh scale={1.5}>
        <sphereGeometry args={[1, 64, 64]} />
        <MeshDistortMaterial
          ref={materialRef}
          color="#1e293b"
          distort={0.15}
          speed={1}
          metalness={0.9}
        />
      </mesh>
    </Float>
  );
}
```

Critical correctness requirements:
- `_targetColor.set(target.color)` — uses `.set()` on a pre-allocated Color instance, NOT `new THREE.Color(target.color)` inside useFrame
- `usePatternStore.getState()` is called inside useFrame — the non-reactive direct read. This is the R3F-recommended pattern to avoid re-renders.
- `usePatternStore((state) => state.target.floatIntensity)` and floatSpeed ARE reactive selectors — they live outside useFrame and correctly cause <Float> to re-render only on domain changes
- The component signature is `export function ShimmerCore()` with no parameters
- Initial material props (color, distort, speed, metalness) match DEFAULTS from patternStore
- Import order: React → @react-three packages → three → internal (per CONVENTIONS.md)
