## Phase 00: Level 0: Bunker Reconstruction

**Goal:** Prove the core fun loop before expanding modules: real household
actions rebuild a destroyed base while chaos attacks it.

**Plans:** 4 plans

Plans:

- [x] `00-01-PLAN.md` — Consolidate Phase 0 product/spec docs and fill empty fitness/ML source specs.
- [x] `00-02-PLAN.md` — Define behavior-event data model, screenshot ingestion contracts, wipe data, kill switch stub, and migrations.
- [x] `00-03-PLAN.md` — Build the native Level 0 Bunker vertical slice with modes, degradation, locked door, Intel Drop, and chaos event.

### Phase 00 Requirements

- `P0-SPEC-CONSOLIDATION` — Phase 0 docs consistently describe AI-first Bunker Reconstruction.
- `P0-NO-HARDCODED-WORKOUTS` — Fitness is adaptive support, not a fixed workout foundation.
- `P0-BEHAVIOR-EVENTS` — Every meaningful user action is logged from first interaction.
- `P0-WIPE-DATA` — Full data deletion is real and testable.
- `P0-KILL-SWITCH-STUB` — Hardcoded kill switch contract exists before growth work.
- `P0-SCREENSHOT-INGESTION` — Screenshot upload/storage path exists before parsing.
- `P0-BUNKER-VERTICAL-SLICE` — Native Level 0 loop is playable.
- `P0-VISUAL-MODES` — Military, Ethereal, and Mixed modes are represented.
- `P0-CHAOS-EVENT` — Household chaos attacks the base without shaming the user.
- `P0-INTEL-DROP` — One earned `// INTEL RECOVERED` moment exists.
- `P0-JOINT-OPS-FITNESS` — Shared fitness momentum is supported as a Bunker mission.
- `P0-GHOST-OPS` — Quiet participation is supported.
- `P0-THREE-QUESTION-MAX` — Onboarding/planning asks at most three high-yield questions.
- `P0-ADAPTIVE-ACTIONS` — Skip, substitute, shuffle, defer, and correction are first-class behavior signals.

### Phase Gate

Do not start Phase 1 until the Level 0 loop is fun enough that a stressed parent
understands it in under 30 seconds:

> The house is under attack. We are rebuilding the base. Do one real thing and
> the base gets better.
