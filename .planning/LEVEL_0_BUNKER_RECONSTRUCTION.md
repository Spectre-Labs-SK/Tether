# Level 0 — Bunker Reconstruction

**Status:** Canonical jumping-off point from Cade direction, 2026-05-13.

## Core Pitch

The opening loop is not cozy productivity and not avatar armor drops.

The user and their household are rebuilding a destroyed base while life attacks it. Kids, toddlers, teenagers, bills, groceries, sleep debt, chaos, and interruptions become the enemy pressure system. The app should be funny, lighthearted, and emotionally honest about parenting being difficult.

## Level 0 Name

Working title: **Bunker Reconstruction**.

Theme variants:

- **Military:** wrecked tactical bunker, amber/green terminal, debris, gear rack, sealed blast doors.
- **Ethereal:** ruined prism sanctuary, living light, overgrown machinery, strange quiet, door glyphs.
- **Both / Mixed:** roots through concrete, prism light through dust, tactical salvage becoming alive.

## Design Laws

- Fun is a requirement, not polish.
- The base changes because the user did something real.
- Rewards are earned from household survival, not random loot.
- Silent degradation beats shame. The base gets dimmer, dirtier, and quieter; the app does not scold.
- Parenting tone is honest: chaos can be funny without pretending it is blessed or easy.
- Drops should create curiosity, anticipation, and narrative momentum.

## Primary Loop

1. User completes a real task.
2. The base physically improves.
3. A visible future unlock gets closer.
4. Sometimes an earned Intel Drop opens story or mechanics.
5. Household chaos attacks the base and creates lighthearted pressure.
6. User and partner repair, defend, and expand the base together.

## Task-To-Scene Examples

- Drink water -> debris clears from a floor section.
- Brush teeth / shower -> gear rack gets a new item.
- Work block -> terminal brightens and shows more data.
- Exercise -> sandbag, pull-up bar, or training corner appears.
- Sleep goal -> cot gets a blanket or lights dim safely.
- Grocery win -> mess hall shelf restocks.
- Finance win -> vault door stabilizes or envelope crates appear.
- Parenting save -> base defense holds against a chaos event.

## Room Unlocks

Rooms replace generic drops as the core progression reward.

- Armory -> preparedness, personal tasks, fitness later.
- Comms -> reminders, partner coordination, weather/calendar.
- Mess Hall -> groceries, pantry, cooking.
- Vault -> envelopes, safety net, debt snowball.
- War Room -> household planning and executive function automation.

Locked doors must be visible before they open. Anticipation matters.

## Intel Drops

Intel Drops are earned milestones, not random rolls.

- Terminal flashes: `// INTEL RECOVERED`
- Contents: mission briefs, audio fragments, house/base lore, classified notes.
- Unknown until opened.
- Should teach, reward, or reveal, not merely decorate.

## HUD Evolution

- Recruit -> basic tasks and degraded base.
- Operative -> comms log, streaks, weather, small automation.
- Ghost -> cleaner minimal UI, fewer prompts, more confidence.
- Spectre -> ethereal/mixed unlocks and deeper base evolution.

## Household Chaos / Base Attack Layer

The most important fun correction: the enemy is not the user. The enemy is chaos.

Babies, toddlers, teenagers, bills, appointments, groceries, missed sleep, school forms, and partner overload can become playful base-attack events. The point is mood transformation: "we are under attack, defend the base" is more fun than "please complete your wellness routine."

Examples:

- Toddler breach: snack shelf hit, mess hall needs repair.
- Teenager drama wave: comms overloaded, restore signal with one small task.
- Baby night raid: sleep system damaged, next day tasks degrade automatically.
- Bill impact: vault shield flickers, move money manually to stabilize.
- Grocery shortage: mess hall alarm, generate list or swap meal.

## Skeptical Notes

- The pasted mockup proves the mood, but it is not the architecture yet.
- Avatars and armor drops are weaker than base reconstruction unless they directly affect rooms or household comedy.
- A beautiful bunker with fake tasks is a trap. The task engine and behavior logging must be real from day one.
- A huge animated base can eat months. Level 0 should use a small set of scene states and deterministic unlocks first.

## Phase 0 Build Target

Build one vertical slice:

1. Level 0 scene with Military/Ethereal/Mixed switch.
2. Silent degradation states.
3. Task completion updates the base.
4. One visible locked door.
5. One earned Intel Drop.
6. One chaos/base-attack event.
7. Behavior events logged for every task, skip, shuffle, completion, and event response.

If this is not fun in three weeks, the product is pointing wrong.
